# Roundtable Frontend Contract

> Updated: 2026-05-16

## Purpose

This note captures the current contract between the local backend and the Lovable weekly roundtable page.

## Public vs Operator Access

Public GET endpoints:

- `GET /health`
- `GET /weekly/current`
- `GET /weekly/archive`
- `GET /weekly/{week_key}`
- `GET /session/{session_id}`
- `GET /api/replay/{session_id}`

Protected mutating endpoints:

- `POST /session/start`
- `POST /session/message`
- `POST /session/intervene`
- `POST /session/shock`
- `POST /session/{session_id}/summary`
- `POST /session/{session_id}/wiki-proposals`

Protected routes require:

- header: `X-Roundtable-Token`
- backend env var: `ROUNDTABLE_OPERATOR_TOKEN`

## Weekly Archive Contract

Featured weekly session:

- `GET /weekly/current`

Archive list:

- `GET /weekly/archive?limit=12&offset=0`

Single weekly preview:

- `GET /weekly/{week_key}`

Full replay payload:

- `GET /api/replay/{session_id}`

## Live Session Start Contract

To avoid an empty chamber state, Lovable should start sessions with:

```json
{
  "prompt": "...",
  "actors": ["china", "us", "eu"],
  "mode": "debate",
  "include_shared": true,
  "auto_generate_opening_turn": true
}
```

If `messages` is returned from `POST /session/start`, the frontend should append them immediately to the transcript before waiting for `POST /session/message`.

## Snapshot Fallback

Static snapshots are generated by:

- `node scripts/snapshot-roundtable.mjs`

Output:

- `public/roundtable-archive/current.json`
- `public/roundtable-archive/archive.json`
- `public/roundtable-archive/replay/<session_id>.json`

Frontend fallback strategy:

1. try live backend
2. if network failure, fall back to snapshot JSON
3. do not fall back for mutating POST actions

## Propaganda Lab Contract

`propaganda-lab` can emit structured poster-dialogue metadata.

For live turns:

- `message.content` contains a compact text payload
- `message.metadata` contains poster fields

For replay turns:

- `event.content` contains the compact text payload
- `event.metadata` contains poster fields

Poster metadata keys:

- `format`
- `slogan`
- `commentary`
- `image_prompt`
- `image_url`
- `image_provider`
- `image_model`
- `image_status`
- `response_target`
- `intended_image_stack`

Frontend should render poster-dialogue turns as image-first propaganda cards rather than normal transcript bubbles.
